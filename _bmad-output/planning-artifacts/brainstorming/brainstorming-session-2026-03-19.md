## Phase 1：What If Scenarios 发散产出

### 核心洞见

**[洞见 #1]: 书签是哑的**
_Concept_：现有书签只记录 URL 和标题，完全不记录用户的思维上下文。"为什么收藏这个"是最有价值的信息，但没有任何工具记录它。
_Novelty_：从"存了什么"到"想到了什么"，这是根本性的产品范式升级。

**[洞见 #2]: 时间轴 vs 文件夹**
_Concept_：传统书签用文件夹分类，但人类记忆是基于时间和情境的。同一时期收藏的内容放在一起，比文件夹分类更能还原"那段时间你在关心什么"。
_Novelty_：书签集合 = 一段学习旅程的切片，而不仅仅是信息碎片的堆砌。

**[洞见 #3]: Readwise 的盲区**
_Concept_：Readwise 记录"你划线了什么"，但不记录"你当时的想法是什么"。这个空白正是产品机会。
_Novelty_：比 Readwise 更深一层——不只是内容摘录，而是思维轨迹。

**[洞见 #4]: C 是 B 的进阶功能**
_Concept_：社交注解（带视角分享书签给好友）依赖于用户已经养成"存书签+写注解"的习惯。没有 B 就没有 C，B 是地基。
_Novelty_：产品路径自然生长，不是功能堆砌。

**[洞见 #5]: 先 B2C 后 B2B**
_Concept_：科研团队是最高付费意愿的用户，但必须先用个人用户验证核心习惯，再向团队扩张（Notion/Figma 路径）。
_Novelty_：降低冷启动风险，用个人口碑驱动团队采购。

## Phase 2：形态学分析 产出

### 产品维度矩阵

| 维度 | 选项 A | 选项 B | 选项 C |
|---|---|---|---|
| **① 捕捉方式** | 浏览器插件（主动存）✅ | 被动追踪（停留时长）| 移动端扫一扫 |
| **② 注解类型** | 一句话理由（"为什么存"）✅ | 情绪标签（感受）| 关联问题（"这让我想到…"）✅ |
| **③ AI 能力** | 自动聚类分类 ✅ | 生成知识演变摘要 ✅ | 主动推送遗忘内容 |
| **④ 时间呈现** | 时间轴视图 ✅ | 知识图谱（节点连线）✅ | 月度/季度回顾报告 |
| **⑤ 社交层** | 无（纯个人）✅ MVP | 带注解分享给好友 ✅ V2 | 团队共享知识库 V3 |
| **⑥ 商业模式** | 个人免费 + Pro 订阅 ✅ | 团队按座位收费 | 一次性买断 |

### 三个组合方案

**组合 1 — MVP（优先执行）**
> 插件 + 一句话注解 + 自动聚类 + 时间轴 + 纯个人 + 个人订阅
> *目标：验证"存书签+写注解"的核心用户习惯*

**组合 2 — Thoughtmark 完整版（目标形态）**
> 插件 + 关联问题注解 + 知识演变摘要 + 知识图谱 + 好友分享 + 个人订阅
> *定位：第二大脑 + 知识社交，差异化最强*

**组合 3 — 科研团队版（V3 扩张）**
> 插件 + 关联问题注解 + 自动聚类 + 季度报告 + 团队知识库 + 按座位收费
> *定位：学术研究团队文献知识管理，替代 Zotero*
