### 验证结果

- ✅ `tsc --noEmit` — 0 错误
- ✅ `npm run dev` — WXT 0.20.20 dev server 启动成功

### 踩坑记录

1. **WXT v0.20.20 npx init 失败**：`npx wxt@latest init` 报 `rolldown MODULE_NOT_FOUND`（原生绑定问题），即使 Node 22 也不行。但 `npx wxt@0.19 init --template react --pm npm` 成功，安装后 postinstall 的 `wxt prepare`（WXT 0.20.20）正常工作。问题出在 npx 临时环境，不是包本身。
2. **nvm install 22 arm64 问题**：nvm-windows 默认尝试 arm64，需要 `nvm install 22.20.0 64` 显式指定 64-bit。
3. **nvm use 需要管理员权限**：在 IDE 终端中挂起，必须在 Administrator 终端执行。

### 关键设计决策

- WXT 模板的 `entrypoints/` 目录直接对应 architecture.md 的结构，无需额外迁移
- `lib/api.ts` 提供 `apiFetch`/`apiGet`/`apiPost` 三层封装，后续 Story 只需 import 使用
