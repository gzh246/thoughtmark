- JWT session 策略：适合插件端共享认证（无服务端 session 依赖）
- Middleware 用 `getToken()` 而非 `auth()` wrapper：兼容 Next.js 16

## Story 3.3: 离线暂存与自动同步

**执行日期**：2026-03-20
**执行工具**：Antigravity (Gemini)
**会话**：`13b99321-3b0f-4c35-91ba-0a2fc971053f`
**Git Commit**：`0a65722`

### 改动文件表

| 文件 | 改动类型 | 说明 |
|---|---|---|
| `thoughtmark-extension/lib/storage.ts` | REWRITE | Chrome Storage 离线队列完整实现（100条/7天） |
| `thoughtmark-extension/lib/sync.ts` | REWRITE | 同步逻辑（逐条上传 + 字段映射 + onMessage 监听） |
| `thoughtmark-extension/entrypoints/background.ts` | REWRITE | Service Worker（自动同步 + 安装检查 + 唤醒检查） |
| `thoughtmark-extension/entrypoints/popup/App.tsx` | MODIFY +30 | 网络失败 → 离线保存 + 📦 暂存提示渲染 |
| `thoughtmark-extension/entrypoints/popup/App.css` | MODIFY +12 | `.offline-message` / `.offline-count` 样式 |

### 核心功能点

- **离线队列**：Chrome Storage 队列，最多 100 条，7 天过期自动清理
- **自动同步**：Service Worker 唤醒时检测在线状态 → 自动上传队列
- **消息触发**：Popup 可通过 `runtime.sendMessage({ type: 'SYNC_OFFLINE_QUEUE' })` 手动触发
- **字段映射**：`annotation → whySaved`, `tags → quickTags`（sync 上传时转换）
- **降级提示**：网络失败 → 📦"已暂存，待网络恢复后同步"（2秒后关闭弹窗）

### 验证结果

- ✅ Extension `tsc --noEmit` — 0 错误
- ✅ Web `tsc --noEmit` — 0 错误
- ✅ GitHub push — commit `0a65722`

### 关键设计决策

- **逐条上传**而非批量：任一条失败立即停止，保留剩余待同步，避免数据丢失
- MV3 Service Worker `addEventListener('online')` 不可靠，改用 `runtime.onMessage` + 唤醒时主动检测
- 离线暂存弹窗延长到 2 秒关闭（比成功的 1 秒多，给用户更多时间注意到"暂存"状态）
